# FA Local Boundaries

FA Local owns:

- requester-trust-gated execution intake
- approval posture resolution
- capability admission checks
- bounded execution-plan validation
- truthful execution status and review handoff
- minimized local forensics for execution truth

FA Local does not own:

- application business semantics
- semantic interpretation or planning
- durable workflow memory
- hidden orchestration
- unbounded plugin execution

Cross-repo note: `forge-local-runtime` remains the doctrine and shared-vocabulary authority. This repo implements FA Local within those bounds.
